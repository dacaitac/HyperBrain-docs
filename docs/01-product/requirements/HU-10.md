# HU-10 — Sync PostgreSQL → Notion

**Como** usuario, **necesito** que los cambios realizados por el motor de IA en la base de datos se propaguen automáticamente a mis bases de datos de Notion, **para** visualizar el estado actualizado sin intervención manual.

## Criterios de Aceptación

1. El SyncServ consume el mensaje de propagación desde SQS.
2. Verifica el `source_system` del evento para evitar rebotes (Loop Protection).
3. Propaga el cambio al satélite correspondiente vía API REST de Notion.
4. Actualiza `last_known_checksum` y `last_synced_at` en `sync_mappings`.
