# HU-06 — Registro de Transacciones

**Como** usuario, **necesito** registrar transacciones financieras (ingresos, gastos, transferencias) vinculadas opcionalmente a tareas o proyectos, **para** tener visibilidad del costo real de cada iniciativa y un flujo de caja preciso.

## Criterios de Aceptación

1. El usuario puede registrar transacciones de tipo `INCOME`, `EXPENSE` o `TRANSFER`.
2. Las transacciones soportan partida doble (`origin_account_id`, `destination_account_id`).
3. Las compras con tarjeta de crédito (Pasivo) no afectan el saldo de cuentas Activo.
4. El pago de tarjeta se registra como `TRANSFER` de Activo a Pasivo.
5. Las transacciones pueden vincularse a un `core_executable` y/o `core_cycle`.
