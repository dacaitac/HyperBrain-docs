# HU-07 — Control Presupuestario

**Como** usuario, **necesito** que el sistema me alerte cuando un gasto exceda el presupuesto asignado a una categoría, **para** evitar desviaciones y mantener el control del flujo de caja.

## Criterios de Aceptación

1. Los presupuestos operan con rangos de fechas flexibles (`fecha_inicio`, `fecha_fin`).
2. Al registrar una transacción, el sistema valida contra el presupuesto activo de la categoría.
3. Se emite una alerta si el gasto acumulado supera el `limit_amount` del presupuesto.
4. Los presupuestos pueden configurarse con política de arrastre (acumular sobrante) o reseteo (base cero).
